]]>https://bleumont.github.io/got-the-basics-now-what/5ff63e1beff6c22f3853bd5cThu, 07 Jan 2021 00:06:42 GMTAfter studying the base of multiple technologies, I'm getting on the way of making my own projects.

I want to start a career on backend development; therefore, these projects need to involve servers, databases, data exchange, authentication, APIs and all that. So, I thought "OK, APIs are interesting and involve most of the stuff I want to dig onto, let's make one".

With the task defined, sort of, the next question is, what is it for? How will it be used? After a couple of days, I came up with the idea of making an API for serving people, fake people of course. This data surely would come in handy for other future projects and could be a cool tool to have.

I started defining the person schema, modified it a couple of times and ended with this:

 person: {
        dni,
        name,
        surname,
        birth-date,
        adress: { street, city, postal-code, province },
     }

// Dni stands "documento nacional de identidad", the argentinian citizen's ID.

For the storage, I was between mongoDB and MySQL, ended up with the lather for I decided to store 'city' and 'province' as separated entities and I'm more comfortable with dealing with relational data.

The actual data belongs to public Argentinian datasets. The site has speed limits so I downloaded the data on CSV and JSON files.

This datasets came with a lot of useless data (for me at least) so I made a Python script to take the fields I was interested on and discard the rest, this caused the data to shrink significantly and to be easy to work with. The script was made in plain Python (it isn't pythonic and is ugly af, so you'll never see it), but I want to refactor it with pandas for speed and readability.

The data I used for surnames was taken from a file in this repo, the file contains surnames registered on Spain, I couldn't find a dataset for Latin American surnames, at least not one with the length I needed. The surnames are very similar in both regions, so it should be just fine, the only thing that is left out are the native American names, since it's a Spaniard list, it contains mostly European surnames.

One more thing I wanted to add for the people's data was a profile image. For this I will take a picture from this site. It's so cool! I'll not be saving the pictures anywhere; I'll just pass the URL.

I have a lot of work from now on! I have to polish the schema, create and populate the database, make the actual API, read more for implement things like hateoas, test it, make a simple frontend, more tests... I wanna quit already.
